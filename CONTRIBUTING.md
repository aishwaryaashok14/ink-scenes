# Contributing

Thanks for wanting to improve Ink Scenes.

Good contributions are usually small and visual:

- stronger example prompts
- cleaner reference guidance
- better QA checks for generated images
- gallery examples that match the style
- install notes for Claude Code, Codex, or other agent tools

## Style guardrails

Keep the visual language consistent:

- one concept per image, staged as one everyday scene
- flat warm cream background — never pure white, never textured
- fine ink linework; shading by crosshatch and stipple only
- one red-orange accent hue with a single dominant carrier
- quiet earth tones only, always hushed below the accent
- completely wordless — scribble instead of text
- robots are retro, matte, and deadpan — never kawaii, never menacing

Avoid adding infographic language, labels and arrows, extra color systems, or diagram-style layouts.

## Pull requests

Before opening a PR:

1. Keep the change focused.
2. Check that examples still point to real files.
3. If you add gallery images, make sure they pass the QA checklist in `ink-scenes/references/qa-checklist.md`.
4. Mention whether you tested the skill in Claude Code, Codex, or both.

If you are not sure whether an idea fits, open an issue first.
